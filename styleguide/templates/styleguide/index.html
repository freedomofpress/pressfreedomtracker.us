Buttons

The sass for buttons can be found in components/_buttons.sass file. An important thing to remember while using the following styles is whether the HTML markup should have an <a> or <button>. In other words, should the markup appear be a link or a button. The simple rule to follow usually is if the button takes you to another page, then it should be marked up as a link (i.e. <a>) whereas if it changes something in the current page, e.g. filter the content of the page, then it should have the mark up of a button (i.e. <button>)

Category Icons

The symbol and the text are independent of the HTML element. So for instance, the categories with text can be implemented with class as defined below even in a <span>. So for the link to have the same styles as a .text-link, one must add that class as well.

<a class="text-link category category-{{ category.page_symbol }}" href="{% pageurl category %}">{{ category.title }}</a>: {% if category %} {{ category.title }} {% endif %}

Details

Key-Value Details

Use a structure similar to this when creating a key value details table: 

			<dl class="details-table" aria-labelledby="incident-details-title">
				<h3 class="heading-table category category-{{ category.page_symbol }}" id="{{ category.slug }}-details-title">
					{{ category.title }}
				</h3>

				{% for detail in category_detail %}
					{% if detail.html %}
					<div class="details-table__row">
						<dt class="details-table__label">
							{{ detail.name }}
						</dt>
						<dd class="details-table__value">
							{{ detail.html | safe }}
						</dd>
					</div>
					{% endif %}
				{% endfor %}
			</dl>

All different card details have their own template include files. For example, incident details are accessed like: {% include "incident/_details-table.html" with incident=page only %} If a new details table is needed for a specific page, create a new include template file, following above structure
